Acerca de Thymeleaf

¿De dónde proviene Thymeleaf? ¿Está fabricado o respaldado por alguna empresa de software?

Thymeleaf es un software de código abierto creado originalmente por el ingeniero de software español Daniel Fernández y ahora desarrollado y mantenido por un gran equipo de personas excepcionales.

No está fabricado ni respaldado por ninguna empresa de software (ni de ningún otro tipo) y se ofrece al público de forma totalmente gratuita, tanto en formato binario como en código fuente, bajo la Licencia Apache 2.0.

¿Es estable? ¿Está listo para producción?

Sí, todas las versiones estables (no beta) pueden considerarse listas para producción.

¿Thymeleaf ofrece algún tipo de soporte comercial?

No, no lo hace. Actualmente no se ofrece soporte comercial oficial.

¿Quién utiliza Thymeleaf?

Nuestra página '¿Quién utiliza Thymeleaf?' contiene una lista de empresas que utilizan Thymeleaf en sus productos, proyectos, sitios web o incluso lo ofrecen como parte de su portafolio de Servicios Profesionales. También encontrará datos de descarga.

¿Cuál es la política de versiones del proyecto?

Los números de versión de Thymeleaf tienen tres componentes: x.y.z (como: 2.0.1).

  • x es el número de versión principal. Un cambio en este número implica una reescritura del núcleo o la arquitectura, o una cantidad importante de cambios en las API de la biblioteca que podrían afectar la compatibilidad con versiones anteriores o el soporte de versiones anteriores.
  • y es el número de versión secundaria. Un cambio en este número significa que se ha aplicado un conjunto moderado de cambios, ofreciendo nuevas funciones y mejoras sin modificar significativamente las API y estableciendo periodos de desuso para todas las funciones modificadas, a fin de permitir un periodo de adaptación a las nuevas interfaces.
  • z es el número de versión de compilación/corrección de errores. Un cambio en este número significa la corrección de algunos errores o la incorporación de una serie de mejoras de rendimiento o documentación. Las versiones de corrección de errores no deben modificar ninguna API y, por lo tanto, deben ser reemplazos directos de cualquier versión anterior dentro de la misma versión principal y secundaria.

¿Cómo puedo contribuir a Thymeleaf?

La primera y más útil manera de contribuir al proyecto es ayudando a otros usuarios en sitios populares como StackOverflow. Construir una mejor comunidad siempre es la prioridad número uno.

También puedes ayudar a Thymeleaf promocionándolo: escribe en tu blog, tuitea, habla de ello en tu JUG local o simplemente comenta sus ventajas en tu empresa/trabajo y entre tus compañeros desarrolladores. Ayuda a difundirlo. ¿Crees que a tu abuela le podría interesar? ¡Díselo! :-)

Si ya te esfuerzas al máximo para ayudar a los usuarios y promocionar tu producto, y aún crees que podrías hacer más, hay un par de cosas que siempre son útiles:

  • Ayuda con las pruebas. Ninguna cantidad de pruebas es suficiente, y tu experiencia como usuario es extremadamente valiosa para crear pruebas útiles que los desarrolladores de Thymeleaf quizá nunca considerarían. Thymeleaf cuenta con su propia biblioteca de pruebas y su propio repositorio de pruebas, y siempre hay espacio para contribuciones. Si deseas sugerencias sobre qué probar, podemos guiarte hacia las áreas que podrían beneficiarse más de tu ayuda.
  • Ayuda con la documentación. Hay una sección de "Artículos" en la página de documentación que siempre recibe nuevos artículos, especialmente si incluyen un buen ejemplo de aplicación. Tu nombre aparecerá como autor en el artículo para que puedas añadirlo a tu currículum ;), pero, por supuesto, te pediremos un alto nivel de calidad y un buen conocimiento del inglés. Mantenemos una lista de sugerencias de temas aquí, pero siempre puedes proponer los tuyos al equipo.

¿Qué más? ¡Claro que puedes aportar tus propias ideas! Puedes crear una problema en GitHub o enviar un correo electrónico directamente al Equipo. Estaremos encantados de revisar y debatir tus ideas contigo. Por favor, no subas una solicitud de incorporación de cambios con tu función favorita sin consultarlo previamente con el equipo. Thymeleaf es un proyecto complejo que requiere cierta organización, y es posible que no podamos aceptar tu código, incluso si le dedicas mucho esfuerzo (¡o incluso si tu idea es buena!).

Finalmente, tenga en cuenta que todas las contribuciones de artefactos a Thymeleaf (código, documentación, etc.) están reguladas por ciertos Términos y Condiciones . Antes de aceptar cualquier parche, artículo, etc., le solicitaremos que acepte explícitamente estos términos (un correo electrónico o un comentario en GitHub bastarán).

General

¿Es Thymeleaf un framework web?

No, no lo es. Es un motor de plantillas.

Sin embargo, los motores de plantillas desempeñan un papel fundamental en los frameworks web y son uno de sus componentes más importantes, ya que se encargan de producir la interfaz de usuario o capa de visualización (normalmente en formato XHTML o HTML).

Spring MVC, Struts o Apache Wicket son ejemplos de frameworks web, mientras que JSP, Velocity o FreeMarker son ejemplos de motores de plantillas.

¿Qué tipos de plantillas puede procesar Thymeleaf?

Las plantillas de Thymeleaf pueden ser:

  • HTML (HTML5, XHTML 1.0/1.1, HTML 4)
  • XML
  • TEXT (texto plano)
  • JAVASCRIPT (ficheros .js)
  • CSS (ficheros .css)

¿Se puede utilizar Thymeleaf como sustituto completo de JSP y JSTL?

Por supuesto. No solo es posible, sino que te animamos a hacerlo. Consulta nuestro artículo sobre Thymeleaf vs. JSP si quieres saber cómo se comparan estas dos tecnologías.

¿Cómo se compara Thymeleaf con otros motores de plantillas como Velocity o FreeMarker?

Tanto Velocity como FreeMarker son excelentes programas, pero abordan el problema de las plantillas con una filosofía muy diferente a la de Thymeleaf.

Thymeleaf hace especial hincapié en las plantillas naturales, lo que permite que las plantillas se conviertan en prototipos funcionales, algo que los otros dos no permiten. Su sintaxis intenta ser (posiblemente) más limpia y estar más en sintonía con las tendencias actuales en desarrollo web. Además, desde un punto de vista arquitectónico, tanto Velocity como FreeMarker funcionan como procesadores de texto secuenciales, mientras que Thymeleaf se basa en técnicas de análisis de marcado. Esto permite a Thymeleaf aprovechar las interesantes características específicas de los entornos basados ​​en marcado, especialmente la web.

De todas formas, la mejor manera de comparar estas tecnologías es utilizarlas uno mismo y sentir cuál se adapta mejor a uno.

¿Se puede utilizar fuera de aplicaciones web en entornos no web?

Sí, se puede. Aunque Thymeleaf (especialmente sus dialectos estándar) ofrece muchas funciones especialmente útiles en entornos web, puede utilizarse para procesar documentos HTML o XML no web (por ejemplo, XML de datos) u otros tipos de plantillas que no están diseñadas para enviarse por HTTP (por ejemplo, contenido de correo electrónico de texto/HTML).

No uso Spring en absoluto. ¿Puedo seguir usando Thymeleaf?

Por supuesto. Thymeleaf ofrece una buena integración con Spring MVC a través de su dialecto SpringStandard (incluido en los paquetes thymeleaf-spring3, thymeleaf-spring4 y thymeleaf-spring5), pero la integración con Spring es completamente opcional y, de hecho, el dialecto Standard está diseñado para usarse sin Spring.

¿Cómo puedo utilizar una versión SNAPSHOT de Thymeleaf?

De vez en cuando, Thymeleaf publicará una versión preliminar que le permitirá probar funciones nuevas o en desarrollo. Estas versiones no están listas para producción, pero son lo suficientemente estables para la mayoría de las pruebas y le permitirán probar nuevas funciones antes de que estén disponibles para el público general.

Por ejemplo, si 3.1.3.RELEASE es la última versión estable publicada, las instantáneas probablemente aparecerán bajo los números de versión 3.1.4-SNAPSHOT o incluso 3.2.0-SNAPSHOT.

¿Cómo se analiza Thymeleaf?

Thymeleaf cuenta con su propio marco de pruebas, llamado thymeleaf-testing. Este marco de pruebas no solo permite al equipo de Thymeleaf probar fácilmente la biblioteca, sino que también permite a los usuarios probar la capa de vista de sus aplicaciones. Puedes explorarlo en GitHub.

Utilizando este marco de pruebas, Thymeleaf mantiene otro repositorio de GitHub completamente dedicado a las pruebas. Pruebas, pruebas y más pruebas. Se trata del repositorio thymeleaf-tests, que puedes encontrar aquí. ¿Quieres ver las cifras? En junio de 2014, el repositorio thymeleaf-tests contenía más de 1300 archivos de prueba, con un total de más de 37 000 líneas de código de prueba.

Además, utilizamos Travis CI para la integración continua, gracias a lo cual cada versión se publica tras ser probada en entornos sin Spring, Spring 3.0, Spring 3.1, Spring 3.2 y Spring 4.0, con Java 6, 7 y 8 (desafortunadamente, Travis no es compatible con Java 5, por lo que lo probamos manualmente).
